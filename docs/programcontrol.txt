program control testing & design

program must:
- interpret command line
- request file lists and file contents from FileHandler
- pass file contents to CipherService
- return numbered file lists as strings
- return file contents or error messages as strings
- only print output in run()

Classes:
ProgramControl

Methods:
- run(String[] args): interprets arguments, calls appropriate method, prints result
- displayFileList(): returns formatted numbered file list as a String
- displayFileContents(String fileId, String optionalKey): returns file contents as a String, or an error message


Test Cases:
1. displayFileList() with files present:
   - Should request file list from FileHandler
   - Should return formatted string with numbered list (e.g., "01 filea.txt")

2. displayFileList() with empty folder:
   - Should return an empty string

3. displayFileContents() with valid file number:
   - Should request the corresponding file from FileHandler
   - Should pass contents to CipherService with default key
   - Should return deciphered text

4. displayFileContents() with invalid file number:
   - Should return an error message
   - Should never call FileHandler.readFile()

5. displayFileContents() with non-numeric argument:
   - Should return an error message

6. displayFileContents() with file number and custom cipher key:
   - Should request file contents from FileHandler
   - Should pass contents to CipherService with the provided key
   - Should return deciphered text

7. displayFileContents() with missing file:
   - FileHandler throws or returns error
   - Should catch and return error message

8. displayFileContents() with cipher failure:
   - CipherService throws validation error
   - Should catch and return error message
